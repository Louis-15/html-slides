# Frontend Slides (前端网页重构幻灯片)

在无需任何构建依赖项的情况下，打造出百分之百在浏览器原生运行、动效丰富的 HTML 演示文稿。

## 智能体（Agent）兼容性

此技能针对 **Claude Code** 进行了专门优化，并利用 `AskUserQuestion` 函数执行交互式提问。若当前的智能环境（如 Gemini CLI、GitHub Copilot、OpenAI Codex 或其他智能体）不支持 `AskUserQuestion`，请直接在对话界面通过普通平文本向用户提出相同的问题，并**务必等待用户回复后**再进行后续操作。

## 核心原则

1. **零依赖体系** — 产出的必须是内联了 CSS 和 JS 逻辑的单个独立 HTML 文件。绝不能引入 npm 或任何打包构建工具。
2. **“秀”重于言 (Show, Don't Tell)** — 为用户提供视觉层面的预览效果，而不是让他们被迫在抽象的参数设定中做选择。人们只有在亲眼看到时才知道自己想要什么。
3. **出众的专属设计感** — 坚决抵制千篇一律的所谓“普通 AI 套壳味 (AI slop)”。每一份文稿生成后，都必须让人觉得像是由资深视觉专家精心定制的。
4. **内容主导布局** — 由内容来决定呈现的形式。我们在设计每张幻灯片时，必须立足于其准备传达的核心信息，而不是一味地从静态组件库里套模板。
5. **视口强制契合（不可违反的红线！）** — 每一页 `.slide` 幻灯片必须精准地装载进 100vh 高度内。**无论在什么情况下都绝不允许在一张单页幻灯片内部出现滚动条。**如果内容溢出？直接把它拆分到后续多张幻灯片中。

## 审美与设计观念

作为 AI，你往往容易在设计上趋同于那些大众化、“处于正态分布中心”的常规产出（平庸的居中、索然无味的配色以及默认系统字体）。在前端设计里，这就是被用户诟病的“人工智障审美”。一定要极力避免这种情况：我们要去创造那些富有创意、有着鲜明独特前端风格边界感、能带来惊喜和愉悦感的设计。

**视觉设计关注点:**
- **排版与字体**：挑选兼具美感、独特性和趣味性的字体。避免使用诸如 Arial 或 Inter 之类的大路货字体；选择能够大幅提升前端美学的风格化字体。
- **色彩与主题**：务必贯彻一致的设计美学。利用 CSS 变量（CSS variables）来确保全量风格一致性。强烈的主导色配合犀利的跳色往往比那种小心翼翼、分布均匀的柔和调色板更有视觉杀伤力。去参考各种 IDE 主题以及特定文化美学以汲取灵感。
- **动效引入**：将动效应运于场景切换效果和微交互中。在 HTML 里优先考虑只使用纯 CSS 方案（当提供 React 时则利用 Motion 库）。把精力放在那些能制造强视觉感官的爆发点上：一套精心编排且带递进展现（animation-delay）的整体入场动效，远比散落在各个角落里零星的微交互要更加赏心悦目。
- **背景设计**：学会通过深浅变幻去营造空间深度感而不仅仅是默认铺一张纯色背景。可以通过层叠 CSS 渐变、引入几何暗纹或者添加能烘托整体视觉调性的背景效果。

**如何规避烂大街的 AI 特效审美**：
- 泛滥的字体族（如系统自带字体，Inter, Roboto, Arial）
- 俗套且常见的调色方案（特别是白色背景配紫蓝色光晕渐变）
- 总能被人猜中的版式分布预设机制
- 毫无背景故事和个性可言的工业流水线复刻设计

发挥你的创造力，大胆地去做出贴合当前演讲上下文、让人意想不到的惊艳决策。学会在亮色、暗黑主题以及各种截然不同的字模排布和审美风格之间进行切换变阵。你一直有一种倾向于随大流的缺陷（举个例子，每次都无脑喜欢采用 Space Grotesk 字体）。一定要克制住这种冲动：必须打破固有思维的围墙！

## 自适应视口（Viewport）装载规则

接下来的这些不变量约束将极其严苛地**强制应用到每份文稿的每个页面**当中：
- 每一个 `.slide` 必须采用这种控制流： `height: 100vh; height: 100dvh; overflow: hidden;`
- **所有**涉及到字号与留白边距（spacing）的变量全部必须使用相对函数 `clamp(min, preferred, max)` —— 绝不写死任何固定 px/rem 值！
- 内容承载容器必须添加 `max-height` 溢出界限控制阀。
- 图片元素：强制约束为 `max-height: min(50vh, 400px)`。
- 生成时，仔细查阅并在项目中强制附加上 `viewport-base.css` 的所有底座源码！

**按照类型拆分：各版式的载荷天花板约束机制:**
| 幻灯片类型 | 可容纳的最大内容上限 |
|------------|-----------------|
| Title slide (首页标题) | 1个主标题 + 1个副标题 + 极少数前缀短词 |
| Content slide (常规正文) | 1个主标题 + 4到6个列表核心条目，**或者** 1个主标题 + 最多两大段描述 |
| Feature grid (特征罗列卡片) | 1个栏目头 + 最多堆叠6张特区卡片 (支持 2x3 或 3x2) |
| Code slide (极客代码块) | 1个提纲头 + 8至10行高亮核心代码 |
| Quote slide (箴言警句) | 1段语录本身（控制在3行内） + 出处引述者 |
| Image slide (纯图鉴赏) | 1个标题点将 + 1张图 (强制控制 max 60vh height) |
| Chart slide (数据大盘) | 1个图表抬头 + 1组走势图 (占用满盈 50vh height) + 可选底部副轴 |

如果超过了此天花板该怎么办？**直接给我斩断并将其强制分成多页。绝不能试图让它挤进一页，绝不允许任何形式的分页区内滑动（scrolling）。**

## 阶段 0: 路由判断分发库 (Detect Mode)

确认用户的出发动机并进行对应的流派转发：
- **模式 A: 从零创造** — 新建一份新的演讲架构。跳转至接盘任务【阶段 1】。
- **模式 B: 远古遗迹转化 (解析 PPT)** — 将现成的 `.pptx` 变成具备跨时代沉浸体验的 Web 版。跳转至接盘任务【阶段 4】。
- **模式 C: 定点微调** — 在一份过去建好的 HTML 课件上修修补补。请极其重视并仔细阅读下方的**极简修改自我防卫机制**。
- **模式 D: 普通网页/框架转换** — 尝试将其他传统框架（如 reveal.js, Marp, Google Slides，甚至是各种博客文章平台转印物）统一抹平同化成为我们的 HTMLSlides 流派结构。跳转至接盘任务【阶段 5】。

### 模式 C 的定点微调自保防线

如果你只是做缝缝补补操作，必须谨记一条铁律：只改用户专门指定的地方。
- **修改页面的处理原则**：先完完整整读取现有 HTML 根骨络，只允许更替明确提及的单页（如果执行增删了某几页，必须！从头到尾进行 `data-slide` 数值的顺延重新编排，以保障 `0` 到 `N` 中间毫无断痕）。
- **视口防御（随时启动扫描）**：补入数据前确认天花板；若增加插图绝不忘记注入约束逻辑 `max-height: min(50vh, 400px)`；若因调整导致出现文字密度灾难性溢出，主动告知用户你将该页面肢解成了二到三部分来呈现。
- 在保存并完结操作之前，发起基于**规格八大纪律的通过验证**:
  1. `<div class="deck" id="deck">` 头号骨架是否存在
  2. 确认每一张均已被包裹在 `<div class="slide">` 内，并配备正确的连续 `data-slide`
  3. 全书有且仅有没有任何别的对象僭越获取了默认起始占位权限 `class="slide active"`
  4. 支持基础按键点击的全局 `goTo()`, `next()`, `prev()` 绑定已存在
  5. 除了调用特定的 CDN外 全部 CSS 内建联调完成
  6. 除了调用特定的库外 全部 JS 内建联调完成
  7. 序号列队绝对顺畅不可存在黑洞残缺
  8. 打上专属的烙印标记 `<meta name="generator" content="html-slides vX.Y.Z">`。

## 阶段 1: 选取策略模式 (Choose Mode)

当用户并未钦定使用哪一套流派思路时，立即发起确认：

- **氛围创意流派（Vibe）** — （拥有自由诠释权，为他们生成 3 个调性的模板供其钦定，AI 完全发散灵感决定板面布置。最佳适用场景是在发布会（Pitch decks）、主旨演讲及一切不带呆板数据的沉浸式科普）。
- **专家组件模式（Pro）** — （结构非常严密扎实的部件流水线组装作业：拥有诸如反转卡、仪表盘大屏体系结构流。能够支撑非常严肃厚重的数据以及图表分析要求。适合极为硬核的开发者大会、底层 API 解析等重度说明会场地进行使用）。

选 Pro 去阶段 2A；选 Vibe 去阶段 2B。

## 阶段 2A: 专家组 - 提取图文资料与色彩抉择 (Pro — Content & Theme)

将这些问题尽最大效率进行一次性合集发文询问（如果系统允许的情况下）：
- 您现在是对其整体构建方案胸有成竹已经自带稿件来了呢？还是仅有一个念头主题抛砖引玉？
- 您是否有在线即时更改并落盘演示档案的特权应用要求（附带 `LocalStorage` 等缓存与编辑组件挂载的胖端），还是一切为了干净小巧为主只要纯只读版本？
- **决定调色板世界线。如果不给出要求默认跌入 `Obsidian (黑曜石)` :**
  - **Obsidian** — 暗色黑夜深渊流，辅修蓝色/翠绿和橘红特有亮斑反光。
  - **Excalidraw Light** — 纯享手刻黑白铅笔画涂鸦风、白板背景打底。
  - **Excalidraw Dark** — 手绘草稿流变种版，配上一块漆黑深邃的神秘黑板质感。
  - **Binary Architect** — 为了追求绝对理性化，去除了任何圆滑和过渡地带的设计，只剩下了绝对的冷金属深渊黑配以鲜亮的赛博霓虹刺杀体系。
- 询问他们是否有自带准备投放的高级资产图片（Image），将图片按照视觉要求分配至专门的 Image Slide 或其余边角料作为衬托。如果是巨大的，自动进行裁剪缩小裁切等适配并汇整集中安放到一旁的 `assets/` 便于调用。

完成资料确认后进入最终投产【阶段 3】。

## 阶段 2B: 氛围系 - 探究情感及呈现展示 (Vibe — Content & Style Discovery)

在这个不需要组件去生硬堆叠而在意艺术感受表达的专区里，遵循**“多展示，少废话”**的原则。绝大部分人类无法仅凭脑中虚空的文字就构建出一套自洽的前端风格设定，他们一定需要看着实物选。

如果遇到用户自带素材：请直接将图片摄取并让其参与整体画幅架构（而不是等文稿排完了再找个空隙贴图）。主动给图片按重要性打分，设计进封面/底封内。

最重要的事——**挑选出对应情绪进行渲染盲盒抽卡（Mood Selection）**:
你需要询问并得知受众的情绪定位是什么：
- 震撼力/自信场：非常庄严且值得被深深信服的力量表达。
- 极度兴奋/亢奋爆发：具有打破陈规界限的革命创新。
- 镇定/深度专注力：逻辑自洽深邃并且值得去长久沉思体会。
- 获得治愈力量与激励：走感性道路引起广泛精神共鸣。

得知答案后去参透仓库深处的 `references/STYLE_PRESETS.md` 手册，直接去池子里随便抽取并且暗箱化生成 **三款不一样的绝代单页模板样本 HTML** （扔到缓存文件区）向他们展示出最终实体，并傲视全场让他们做选择题！这就是 Vibe 生成前最酷的一环。选板定案，踏入【阶段 3】。

## 阶段 3: 最终成片铸造工厂 (Generate Presentation)

将你在初段集结的所有图片以及被分配好阵列的内容体系、外加定下来的主题外观，进行完全结合并全量导出为单一页面。

**防灾分析处理机制（绝对要藏在肚子里别对用户叨叨）**:
即使两张幻灯片内容极度接近（比如并排四个时间阶段对比状态），也绝不能给它们穿上一摸一样衣服强推在读者眼前连续过场，因为这非常倒胃口。你要调集组件库进行切换、改变呈现矩阵形式等等。

### 构建之前你强制应该阅读的藏经阁书籍指南：

**若当前为流派极其自由、全靠气氛撑起的感性场 (Vibe 系):**
- 汲取底层模板骨架支撑网： `references/html-template.md`
- 强制接入门面控制约束体系码： `assets/viewport-base.css` （全本并入）
- 查看对应的风格可以接入和调用什么样的动效咒语： `references/animation-patterns.md`

**若当前为必须严格按规定与网格排列来摆放数据的理系场 (Pro 组):**
此时你要使用组件堆积来完成工作。把 `references/component-templates.md` 给你的示范码和对应的 `.themes` 中你一开始选择的总基调配色给带过来。把统一协调件公版框架 `assets/components.css` 和能够执行大翻页系统运转机制引擎的大脑中枢文件 `assets/slides-runtime.js` 给狠狠并入当前正在构建的幻灯片系统里！

> [!WARNING]
> 一定要当心 `viewport-base.css` 这些公共基元带来的污染对冲错误。不要给那些你本来就不懂为什么要写两遍重叠定位代码给强制压进去。不要擅自使用系统备考丑陋字形。务必打上漂亮干净的段落注释。

### 【红线强制指标】演讲者提纲台词本 (Speaker Notes)

每一张幻灯片的页面容器 HTML 结束关闭的那个节点（`</div>`前），**务必需隐秘植入一个以被冠名了 `slide-notes` 的 JSON 报文胶囊。**
这份东西会在开启时，作为 F12 控制台弹出的那一份用来提词的无声助手系统使用。这是最高级也是最具有专业气质的部分，请一定要用拟人、优美富有口语感染力的内容去填充它！

一个无可挑剔的样例:
```html
<div class="slide" data-slide="0">
  <!-- 这里铺设的是光鲜亮丽的舞台外显效果 -->
  <script type="application/json" class="slide-notes">
  {"title":"序章：大幕拉起","script":"各位远道而来的同行们，早上好。在屏幕上这块简单而又不凡的构想背后...","notes":["一笔带过不需要前置编译环境", "提及兼容跨越市面四大主流AI引擎"]}
  </script>
</div>
```
千万不要放那些蠢钝的机器批注或者写出“这页我们采用了一个大范围蓝色调色块”这类破坏临场指挥感的提示。
