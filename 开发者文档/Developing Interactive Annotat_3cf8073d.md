	#### 1b. 浮动菜单 vs 现有富文本工具栏的关系
	
	现在项目里已经有一个富文本工具栏（`editor-rich-text.js`），支持文字变色、高亮背景、下划线、删除线等功能。
	
	你的需求里提到 "批注对应的原文内容可以是背景高亮、文字变色、下划线、删除线等样式，这些样式要跟编辑功能中的富文本编辑是统一的"。
	
	这就存在一个关键设计问题：**加批注时对原文应用的格式，和普通编辑时的富文本格式，用的是同一套系统**。那么——
	
	**方案 A：浮动菜单是独立的微型工具条**
	```
	选中文字 → 弹出浮动菜单 [📝加批注] [A变色] [🖍高亮] [U下划线] [S删除线]
	         → 点击「加批注」→ 先对原文应用选中的格式 → 右侧弹出空的批注气泡等输入
	```
	浮动菜单同时承担"选格式"和"加批注"两个职能，一步到位。
	
	**方案 B：浮动菜单只是一个「📝加批注」按钮**
	```
	选中文字 → 弹出浮动按钮 [📝加批注]  
	         → 点击后 → 上方的编辑工具栏里自动聚焦到格式区域（变色/高亮/下划线/删除线）
	         → 用户选格式 → 右侧弹出空的批注气泡等输入
	```
	格式选择复用现有工具栏，浮动按钮只负责"触发加批注"。
	
	**我倾向于方案 A**，理由：
	- 减少鼠标移动路径——选中文字后手不用跑到页面顶部
	- 浮动菜单紧贴选区，上下文感强
	- 跟你截图中 Word 的交互一致
