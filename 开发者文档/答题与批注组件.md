## 一、组件定位

答题与批注组件是 html-slides 教学课件系统的第 14 个 Zone 2 组件。它将答题（选择题/填空题/拖拽连线）与批注讲解两大功能合为一体，适用于阅读理解、七选五、完形填空、作文批改等教学场景。

### 设计原则

- 属于 Zone 2 组件体系，写在 `zone2-content.css` 中，不新建 Zone 文件
- **独占整页**，始终使用 `layout-single` 布局（Zone 1 标题栏照常保留）
- 打破"页内无滚动条"的旧规则，采用**局部隐形滚动条**
- 零依赖（Zero Dependency），纯原生 HTML/CSS/JS
- 所有色彩通过 CSS 变量继承主题，Glassmorphism 风格统一
- 通过现有 `registerStepStrategy()` API 接入步进系统，兼容蓝牙翻页器

```
├────────────────────────────────────────────────────────────┤

│  ┌───────────────────────┬─────────────────────────────┐  │
│  │                       │                             │  │

│  │   独立滚动             │   独立滚动                  │  │

│  │                       ├─────────────────────────────┤  │
│  ├═══ 可拖动分隔条 ═══════┤   ③' 答案解析批注           │  │
│  │                       │   （同时展开时出现）          │  │
│  │   ② 选项/作答区域      │   独立滚动                  │  │
│  │   (qa-answer-panel)   │                             │  │
│  │   独立滚动             │                             │  │
│  │   默认折叠             │                             │  │
│  │                       │                             │  │
│  └───────────────────────┴─────────────────────────────┘  │
│                                                            │
│  ┌────────────────────────────────────────────────────┐    │
│  │  [📝 批注 3/12]  [📋 作答]              控制按钮区  │    │
│  └────────────────────────────────────────────────────┘    │
└────────────────────────────────────────────────────────────┘
```

### 2.2 三个主要区域

#### ① 正文/题干区域（qa-passage）
- 占据组件主体，不可收起
- 有自己的**隐形滚动条**（4px、半透明、hover 时微显）
- 正文中被批注的文字通过 `<span class="text-anchor">` 包裹

#### ② 选项/作答区域（qa-answer-panel）
- 在正文/题干区域的**下方**，默认折叠
- 展开/收起通过控制按钮或快捷键 `F` 控制
- 有自己的隐形滚动条
- 正文与选项之间有**可拖动的分隔条**，用户可上下拖动调整两者比例
- 有一个**全屏按钮**，点击后选项/作答区域直接占满整个 Zone 2（覆盖正文），适合"先看题再看文"的阅读理解答题策略
- 全屏状态下按钮变为"还原"，点击恢复分隔比例
- 实现方式：flex 比例调整（`flex: 0 0 0px` 隐藏正文 + `flex: 1 1 100%` 选项撑满）

#### ③ 批注面板（qa-notes-panel）
- 在正文/题干区域的**右侧**，默认隐藏
- 展开/收起通过控制按钮或快捷键 `D` 控制
- 展开后与左侧形成 2:1 分栏（正文 67%，批注 33%）
- 有自己的隐形滚动条
- 若选项区域和批注面板同时展开，批注面板分成上下两部分：
  - 上半：正文/题干的批注
  - 下半：答案解析的批注（与选项区域的题目一一对应）
  - 上下分隔比例**也可拖动调整**

### 2.3 状态切换表

| 状态 | 正文区域 | 选项区域 | 批注面板 | 滚动策略 |
|------|----------|----------|----------|----------|
| 默认 | 100% 宽 | 隐藏 | 隐藏 | 正文独立滚动 |
| 仅批注展开 | 60% 宽 | 隐藏 | 40% 宽 | 各自独立滚动 |
| 仅作答展开 | 100% 宽（上+下分隔） | 可见、可拖动分隔 | 隐藏 | 各自独立滚动 |
| 二者同时展开 | 60% 宽（上+下分隔） | 可见、可拖动分隔 | 40% 宽（上下分两部分） | 各自独立滚动（最多4个） |
| 选项全屏 | 隐藏（flex: 0） | 100% 全屏覆盖 | 隐藏 | 选项区域独立滚动 |

## 三、批注系统

### 3.1 原文锚点与角标

被批注的文字通过 `<span class="text-anchor">` 包裹，角标以上标形式显示在右上方。

```html
<span class="text-anchor" data-link="note-01" data-step="1"
      style="text-decoration: underline; text-decoration-color: var(--accent-blue);">
  被批注的长难句
  <sup class="note-badge">1</sup>
</span>
```

- `data-link`：批注唯一标识，永不改变、永不递补（自增 ID 机制）
- `data-step`：步进序号，拖拽排序后由 JS 自动重算
- 原文的富文本格式（下划线/高亮/变色/删除线）直接以 inline style 存储在 `text-anchor` 上，复用 `editor-rich-text.js` 的格式系统
- 删除批注时，JS 通过 `data-link` 找到原文锚点，清除 inline style、解包 `<span>`、删除角标

### 3.2 角标智能避让

当多个角标紧邻时（如整句批注和句中某个词的批注），角标**横向并排**显示，如 `¹²³`，不增加行间距。

### 3.3 批注气泡

```html
<div class="qa-note-bubble" data-link="note-01" data-step="1" draggable="true">
  <div class="qa-note-handle">
    <span class="qa-note-step">1</span>
  </div>
  <div class="qa-note-content" data-edit-id="s1-note-01">
    批注内容...
  </div>
</div>
```

#### 气泡的按钮与手柄

| 元素 | 位置 | 说明 |
|------|------|------|
| 序号球（拖拽手柄）| 气泡左侧全高贯通区 | 绿色圆圈显示序号，同时充当拖拽把手。激活态反色（绿底白球→白底绿球）|
| 📌 选中原文 | 气泡右上角（hover 显示） | 编辑模式下点击气泡后显示，自动选中该批注对应的原文 |
| ✖ 删除批注 | 气泡右上角（hover 显示） | 编辑模式下点击气泡后显示，删除批注 + 清除原文格式 |

#### 气泡的显示规则

- **固定列表方案**：气泡始终按 `data-step` 顺序排列在右侧面板中，位置不跳动
- **激活态**：气泡变大（仅 scale 物理缩放）+ 浮起 + 高亮边框 + z-index 提升
- **降噪**：当有任意气泡被激活时，其他所有气泡半透明 + 缩小
- **内容折叠**：内容超过 3 行自动截断（显示 "…"），点击气泡即可展开全部
- **点击交互**：点击气泡任意区域（除悬浮操作小标外）即切换激活态，使用动态索引查询避免拖拽后错位

#### 拖拽排序

- 使用 HTML5 原生 Drag and Drop API
- 拖拽完成后，JS 从上到下重算所有气泡的 `data-step`
- 同时逆向同步原文侧的 `data-step`
- 放映模式和编辑模式**都可拖拽**

### 3.4 SVG 连线

- 在组件最上层铺设透明 SVG 画布（`pointer-events: none`）
- 连线从原文**角标右侧中点**到气泡**序号球左侧中点**，使用贝塞尔曲线
- 坐标通过 `getBoundingClientRect()` 实时计算
- 滚动时通过 `scroll` 事件 + `requestAnimationFrame` 实时重绘连线

#### 连线显示规则

| 触发方式 | 连线条数 | 持续时间 |
|----------|----------|----------|
| 步进（←→键）展开批注 | 1 条（当前批注） | 持久，直到步进到下一个 |
| 鼠标 Hover 原文/气泡 | +1 条（悬停的）| 临时，鼠标离开即消失 |
| 步进 + Hover 同时 | 最多 2 条 | 各自独立管理 |

#### 双向追视

激活批注时，左右两边都执行 `scrollIntoView({ behavior: 'smooth', block: 'center' })`：
- 原文锚点自动滚到可见区域
- 批注气泡也自动滚到可见区域

## 四、答题系统

### 4.1 支持的题型

| 题型 | 交互方式 | 判分显示 |
|------|----------|----------|
| **单选题** | 点选一个选项 | 正确选项旁 ✓（绿），错选旁 ✗（红） |
| **多选题** | 点选多个选项 | 同上 |
| **填空题** | 用户可不输入就直接提交 | 提交后空位中直接显示正确答案 |
| **连线题**（完形填空/七选五） | 将选项拖拽到对应空位（只显示选项序号ABCD，不显示内容） | 空位分前后两段：前段 ✓✗ 判分；若答错，后段浮现正确选项；若答对，后段不再重复显示正确选项 |

### 4.2 答案数据存储

正确答案由 AI 生成时写在 `data-*` 属性中：

```html
<!-- 选择题 -->
<div class="qa-option" data-option="A" data-correct="true">...</div>

<!-- 填空题空位 -->
<span class="qa-blank-slot" data-blank-id="1" data-correct-answer="however">___</span>

<!-- 连线题空位 -->
<span class="qa-blank-slot" data-blank-id="1" data-correct-answer="C">
  <span class="qa-blank-user">___</span>
  <span class="qa-blank-answer"></span>
</span>
```

### 4.3 判分结果动画

- ✓ 和 ✗ 标记以 `scale(0.3) → scale(1)` 的弹入动画渐入
- ✓ 绿色，✗ 红色
- 填空题的正确答案以 `opacity: 0 → 1` 渐入

### 4.4 答题与批注的隔离规则（重要）

**核心规则**：如果 `qa-answer-panel` 内有答题内容，组件初始状态下**所有批注相关 UI 完全隐藏**——假装这篇文章从未被批注过：

- 正文区域的所有批注角标（`note-badge`）不显示
- 正文区域的所有批注富文本格式（下划线/高亮/变色等）不显示
- 批注面板的批注气泡不可见
- 控制按钮区的"📝 批注"按钮显示为不可用状态

**显示条件**：用户提交答案后，所有批注相关 UI 同时浮现（渐入动画），解锁批注讲解功能。

**编辑模式例外**：编辑模式下始终显示所有批注，否则老师无法编辑。

**CSS 状态类**：
- `.has-quiz` — 有答题内容
- `.submitted` — 已提交答案
- 编辑模式由编辑器全局状态控制

## 五、正文与选项的动态比例调整

### 5.1 可拖动分隔条

正文区域与选项/作答区域之间有一个可拖动的分隔条（resize bar）。

- 用户可上下拖动调整两者的高度比例
- 拖动时实时更新，丝滑的 transition 过渡
- 默认比例：正文 60% / 选项 40%

### 5.2 全屏覆盖

选项/作答区域的标题栏上有一个 `[⬆ 全屏]` 按钮：

- 点击后选项区域占满整个 Zone 2，正文被完全覆盖
- 实现方式：`flex: 0 0 0px` 隐藏正文 + `flex: 1 1 100%` 选项撑满
- 全屏后按钮变为 `[⬇ 还原]` 或 `[📄 查看原文]`
- 点击还原 → 恢复之前的分隔比例

### 5.3 批注面板的上下分隔

当选项区域与批注面板同时展开时，批注面板分成上下两部分：
- 上半：正文/题干批注
- 下半：答案解析批注
- **上下分隔比例联动调整**：右侧批注区域的上下分割线，与左侧（正文和选项之间）是**同一条线**。拖动分隔条时，左右两个区域的高度会**同时改变**，保持对齐。

## 七、添加批注的交互（编辑模式专属）

### 7.1 触发方式

在编辑模式下，用户在正文/题干区域**选中文字**后，文字旁边弹出一个浮动工具条（专属于该组件的上下文菜单，独立于顶部编辑工具栏）。

### 7.2 浮动工具条结构

```
┌─────────────────────────────────────────────────────┐
│  添加批注   [A变色] [🖍高亮] [U̲下划线] [S̶删除线]     │
└─────────────────────────────────────────────────────┘
```

- **"添加批注"**：文字标签，不是按钮，起引导说明作用
- **四个格式按钮**：点击任意一个按钮 = **同时**对原文应用该格式 + 创建新批注
- 将"选格式"和"创建批注"合并为一步操作

### 7.3 操作流程

1. 在编辑模式下，选中正文中的一段文字
2. 选区旁弹出浮动工具条
3. 点击任意格式按钮（如 `[U̲下划线]`）
4. 原文被包裹为 `<span class="text-anchor">` 并应用下划线格式
5. 自动生成新的角标（取当前最大 `data-link` 编号 +1）
6. 右侧批注面板自动添加一个空的气泡，光标自动聚焦进去等用户输入

### 7.4 格式函数复用

浮动工具条里的格式按钮调用的是 `editor-rich-text.js` 中已有的同一套格式化函数，但**额外触发"创建批注"的逻辑**。顶部编辑工具栏依然正常工作（加粗/斜体/字号等普通编辑功能）。

### 7.5 浮动工具条的消失时机

- 用户点击了格式按钮 → 完成操作 → 消失
- 用户点击页面其他位置（取消选区）→ 消失
- 用户修改了选区范围 → 重新定位到新选区

## 八、数据结构

### 8.1 原文锚点

```html
<span class="text-anchor" data-link="note-03" data-step="2"
      style="text-decoration: underline; text-decoration-color: var(--accent-blue);">
  被批注的文字
  <sup class="note-badge">2</sup>
</span>
```

### 8.2 批注气泡

```html
<div class="qa-note-bubble" data-link="note-03" data-step="2" draggable="true">
  <div class="qa-note-handle">
    <span class="qa-note-step">2</span>
  </div>
  <div class="qa-note-content" data-edit-id="s1-note-03">
    批注内容，老师的讲解...
  </div>
</div>
```

### 8.3 关联机制

- `data-link`：批注唯一标识（自增 ID，永不改变、永不递补）
- `data-step`：步进序号（拖拽排序后由 JS 重算）
- 原文格式以 inline style 存在于 `text-anchor` 上
- 删除批注时：JS 通过 `data-link` 找到原文锚点 → 清除 inline style → 解包 `<span>` → 删除角标 → 删除气泡 → 重算序号

### 8.4 预留字段

- `data-note-type`：批注分类标签（第一版不使用，预留给后续"分类步进"功能）

## 九、技术整合

### 9.1 步进系统整合

通过现有 `registerStepStrategy()` API 注册新策略，无需修改 `slides-runtime.js` 核心逻辑。

但需要**小幅改造** `stepForward()`/`stepBackward()` 以支持"一个组件多步"：现有组件（flip/collapse）都是"一个组件一步"，批注组件是"一个组件 N 步"（每个批注是一步）。`hasNextStep()` 接口已预留，只需在步进函数中增加多步检查逻辑。

### 9.2 滚轮拦截

组件内的可滚动区域使用 `data-scrollable` 属性标记，利用现有 `slides-runtime.js` 的智能滚轮拦截。

### 9.3 空格键

空格键用于组件内正文区域的页内滚动，利用现有 `handleSpaceKey()` 函数。

### 9.4 导出行为

Ctrl+S 导出时，清除所有答题状态（用户选择、✓✗ 标记），恢复为空白试卷。批注数据保留。
