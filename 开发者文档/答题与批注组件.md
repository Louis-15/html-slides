> **组件编号**：Zone 2 第 14 号组件

## 一、组件定位

答题与批注组件是 html-slides 教学课件系统的第 14 个 Zone 2 组件。它将答题（选择题/填空题/拖拽连线）与批注讲解两大功能合为一体，适用于阅读理解、七选五、完形填空、作文批改等教学场景。

### 设计原则

- 属于 Zone 2 组件体系，写在 `zone2-content.css` 中，不新建 Zone 文件
- **独占整页**，始终使用 `layout-single` 布局
- **沉浸式逃逸（Immersive Override）**：通过 CSS `:has()` 伪类自动隐藏 Zone 1 标题栏，使 Zone 2 霸占全页高度（右上角品牌 Logo 为全局 `position: fixed` 元素，不受影响）
- **左中右三栏水平布局**，彻底取消上下分隔条
- 打破"页内无滚动条"的旧规则，采用**局部隐形滚动条**
- 零依赖（Zero Dependency），纯原生 HTML/CSS/JS
- 所有色彩通过 CSS 变量继承主题，Glassmorphism 风格统一
- 通过现有 `registerStepStrategy()` API 接入步进系统，兼容蓝牙翻页器

## 二、区域划分与布局

### 2.1 整体结构

```
┌────────────────────────────────────────────────────────────┐
│  Zone 1: Header Bar (标题栏)                                │
│  ⚠ 沉浸式逃逸：当 slide-content 内包含 .quiz-annotation    │
│    时，Zone 1 自动隐藏，Zone 2 获得全页高度                  │
│    右上角 Logo (.branding) 为 fixed 定位，不受影响           │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  ┌─────────────┬──────────┬──────────────────┐             │
│  │             │          │                  │             │
│  │  ① 正文     │ ② 批注   │  ③ 答题          │             │
│  │  (passage)  │ (notes)  │  (answers)       │             │
│  │             │          │                  │             │
│  │  宽度: 5    │ 宽度: 2  │  宽度: 3         │             │
│  │  独立滚动   │ 独立滚动 │  独立滚动         │             │
│  │             │ 默认隐藏 │                  │             │
│  │             │          │                  │             │
│  └─────────────┴──────────┴──────────────────┘             │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

**不设独立控制按钮区**。批注面板的展开/收起入口集成在栏间分割线上（见 2.4 节）。

### 2.2 三个主要区域

#### ① 正文/题干区域（qa-passage）— 左栏

- 占据组件左侧，始终可见，宽度占比 **5/10**（批注隐藏时占 **5/8**）
- 有自己的**隐形滚动条**（4px、半透明、hover 时微显）
- 正文中被批注的文字通过 `<span class="text-anchor">` 包裹
- **非对称 Padding**：左侧无内边距（交给 `.slide` 的 `--slide-padding` 统管），右侧保留内边距（与中间栏的视觉分隔间距）

#### ② 批注面板（qa-notes-panel）— 中栏

- 位于正文与答题之间的**中间栏**，宽度占比 **2/10**
- **默认隐藏**，展开/收起通过分割线悬浮按钮或快捷键 `D` 控制（见 2.4 节）
- 隐藏时，左栏和右栏以 **5:3** 的比例瓜分组件全部宽度
- 展开时，三栏以 **5:2:3** 的比例并存
- 有自己的隐形滚动条
- **双向关联**：每个批注气泡可同时关联左侧的正文锚点和右侧的答题条目
- **展开态栏头**：顶部显示 `📝 批注 3/12`（进度指示）+ 折叠按钮

#### ③ 选项/作答区域（qa-answer-panel）— 右栏

- 占据组件右侧，宽度占比 **3/10**（批注隐藏时占 **3/8**）
- **条件显示**：当右栏内有答题内容时始终可见，无需用户手动展开；当无答题内容时（纯文章解析场景），右栏不显示
- 有自己的隐形滚动条
- **非对称 Padding**：右侧无内边距（交给 `.slide` 的 `--slide-padding` 统管），左侧保留内边距（与中间栏的视觉分隔间距）
- 答题条目也可以拥有角标（`note-badge`），当一道题对应多个正文批注时，角标**横向并排**显示

### 2.3 状态切换表

| 场景 | 状态 | 左栏（正文） | 中栏（批注） | 右栏（答题） | CSS Grid |
|------|------|-------------|-------------|-------------|----------|
| 有答题内容 | 批注隐藏 | 5fr | 隐藏 | 3fr | `5fr 3fr` |
| 有答题内容 | 批注展开 | 5fr | 2fr | 3fr | `5fr 2fr 3fr` |
| 纯文章解析 | 批注隐藏 | 1fr（占满） | 隐藏 | 不存在 | `1fr` |
| 纯文章解析 | 批注展开 | 2fr | 1fr | 不存在 | `2fr 1fr` |

> **注**：不再有"选项折叠""选项全屏""上下分隔条""作答按钮""控制按钮区"等旧概念。右栏的存在与否由 HTML 内容决定（有答题 DOM 则显示，无则不显示），批注展开/收起是唯一的用户可控维度。

### 2.4 栏间分割线与批注入口

相邻两栏之间有一条**极细的半透明分割线**（`1px solid rgba(255,255,255,0.08)`），作为视觉边界。

#### 批注悬浮入口（批注隐藏时）

当批注面板处于隐藏状态时，用户鼠标移动到**分割线附近一定宽度范围内**（约 ±20px），一个 📝 小按钮会自动**浮现在鼠标所在位置的水平线与分割线的交点上**，跟随鼠标 Y 轴移动。点击即展开批注面板。

```
┌──────────┬──────────────────┐
│          │                  │
│          │                  │
│        ──📝──               │  ← 鼠标 hover 到分割线附近，按钮浮现
│          │                  │
│  正文    │  答题             │
│          │                  │
└──────────┴──────────────────┘
```

#### 批注栏头（批注展开时）

批注面板展开后，中栏顶部显示一个**固定栏头**：
- 📝 图标 + 批注计数（如 `3/12`）
- 折叠按钮 `[<<<]`，点击收起批注面板，回到悬浮入口状态

## 三、批注系统

### 3.1 原文锚点与角标（左栏 — 正文侧）

被批注的文字通过 `<span class="text-anchor">` 包裹，角标以上标形式显示在右上方。

```html
<span class="text-anchor" data-link="note-01" data-step="1"
      style="text-decoration: underline; text-decoration-color: var(--accent-blue);">
  被批注的长难句
  <sup class="note-badge">1</sup>
</span>
```

- `data-link`：批注唯一标识，永不改变、永不递补（自增 ID 机制）
- `data-step`：步进序号，由中间栏气泡的排列顺序决定，拖拽排序后由 JS 自动重算
- **统一序号**：左栏角标、中栏气泡序号球、右栏角标使用**同一套序号**，全部由中间栏气泡的上下排列顺序（`data-step`）决定
- 原文的富文本格式（下划线/高亮/变色/删除线）直接以 inline style 存储在 `text-anchor` 上，复用 `editor-rich-text.js` 的格式系统
- 删除批注时，JS 通过 `data-link` 找到原文锚点，清除 inline style、解包 `<span>`、删除角标

### 3.2 答题条目锚点与角标（右栏 — 答题侧）

答题区域的题目/选项也可以承载角标，表示该题目与哪些批注相关联。

```html
<div class="qa-option" data-option="A" data-correct="true">
  <span class="answer-anchor" data-link-answer="note-01">
    <sup class="note-badge">1</sup>
  </span>
  <span class="answer-anchor" data-link-answer="note-03">
    <sup class="note-badge">3</sup>
  </span>
  选项文本...
</div>
```

- 当一道题对应左侧的**多处正文**时，右侧的角标**横向并排**显示（如 `¹²³`）
- `data-link-answer`：指向中间栏的批注气泡，与左侧的 `data-link` 对应同一个批注

### 3.3 角标智能避让

当多个角标紧邻时（如整句批注和句中某个词的批注），角标**横向并排**显示，如 `¹²³`，不增加行间距。左栏和右栏均遵循此规则。

### 3.4 批注气泡（中栏）

```html
<div class="qa-note-bubble" data-link="note-01" data-link-answer="q1" data-step="1" draggable="true">
  <div class="qa-note-handle">
    <span class="qa-note-step">1</span>
  </div>
  <div class="qa-note-content" data-edit-id="s1-note-01">
    批注内容...
  </div>
</div>
```

- `data-link`：关联**左侧正文**的锚点
- `data-link-answer`：关联**右侧答题**的条目（可选，若只是纯正文批注则无此属性）
- 同一个批注气泡可**同时拥有两个关联**，实现双向连线

#### 气泡的按钮与手柄

| 元素 | 位置 | 显示条件 | 说明 |
|------|------|----------|------|
| 序号球（拖拽手柄）| 气泡左侧全高贯通区 | 始终 | 绿色圆圈显示序号，同时充当拖拽把手。激活态反色（绿底白球→白底绿球）|
| 📌 选中原文 | 气泡右上角 | 编辑模式 hover | 自动选中该批注对应的原文 |
| ✖ 删除批注 | 气泡右上角 | 编辑模式 hover | 删除批注 + 清除原文格式 + 清除右栏角标 |
| 🔗 关联左侧 | 气泡右上角 | 编辑模式 hover，且当前气泡**尚未关联左侧**内容时 | 进入关联模式（见第七节） |
| 🔗 关联右侧 | 气泡右上角 | 编辑模式 hover，且当前气泡**尚未关联右侧**内容时 | 进入关联模式（见第七节） |

#### 气泡的显示规则

- **固定列表方案**：气泡始终按 `data-step` 顺序排列在中间面板中，位置不跳动
- **激活态**：气泡变大（仅 scale 物理缩放）+ 浮起 + 高亮边框 + z-index 提升
- **降噪**：当有任意气泡被激活时，其他所有气泡半透明 + 缩小
- **内容折叠**：内容超过 3 行自动截断（显示 "…"），点击气泡即可展开全部
- **点击交互**：点击气泡任意区域（除悬浮操作小标外）即切换激活态，使用动态索引查询避免拖拽后错位

#### 焦点外扩机制

由于中间栏仅占 2/10 宽度（约 192px @1920px），空间紧凑。当气泡被激活（焦点状态）时：

- 气泡通过 `overflow: visible` + `position: relative` + `z-index` 提升，**打破中间栏的宽度限制**
- 向左侧外扩：侵入左栏的**右侧内边距区域**（视觉分隔间距，非正文内容区）
- 向右侧外扩：侵入右栏的**左侧内边距区域**（视觉分隔间距，非答题内容区）
- **严格不覆盖左右两栏的实际内容**，只利用两侧的 padding 空白带
- 外扩距离 = 左栏右padding + 右栏左padding（共约 40-56px），使气泡总可用宽度约 230-250px

#### 拖拽排序

- 使用 HTML5 原生 Drag and Drop API
- 拖拽完成后，JS 从上到下重算所有气泡的 `data-step`
- 同时逆向同步左侧原文和右侧答题的 `data-step`
- 放映模式和编辑模式**都可拖拽**

### 3.5 双向 SVG 连线

- 在组件最上层铺设透明 SVG 画布（`pointer-events: none`）
- **左侧连线**：从正文角标右侧中点 → 批注气泡序号球左侧中点，使用贝塞尔曲线
- **右侧连线**：从批注气泡序号球右侧中点 → 答题角标左侧中点，使用贝塞尔曲线
- 若批注气泡同时关联了左右两侧，则**同时绘制两条连线**
- 坐标通过 `getBoundingClientRect()` 实时计算
- 滚动时通过 `scroll` 事件 + `requestAnimationFrame` 实时重绘连线

#### 连线显示规则

| 触发方式 | 连线条数 | 持续时间 |
|----------|----------|----------|
| 步进（←→键）激活批注 | 1 或 2 条（当前批注的双向连线） | 持久，直到步进到下一个 |
| 点击气泡激活批注 | 1 或 2 条 | 持久，直到点击其他气泡或取消激活 |
| Hover 气泡（非激活态的）| +1 或 +2 条（悬停的）| 临时，鼠标离开即消失 |
| 步进 + Hover 同时 | 最多 4 条 | 各自独立管理 |

> **注**：hover 连线仅在**鼠标悬停气泡**时触发，不监听左栏原文锚点或右栏答题锚点的 hover。

#### 锚点出屏边缘钉定

当某一栏被用户手动滚动，导致已激活批注的锚点滚出该栏的可见范围时：

- 连线端点**钉定在该栏的边缘**（上边缘或下边缘）
- 端点处变成一个小**箭头**，指向上方（`↑`）或下方（`↓`），暗示"关联内容在上面/下面"
- 用户点击箭头或重新激活该批注时，执行 `scrollIntoView` 将锚点重新滚入可见区域

#### 双向追视

激活批注时，左中右三个区域都执行 `scrollIntoView({ behavior: 'smooth', block: 'center' })`：
- 左栏：原文锚点自动滚到可见区域
- 中栏：批注气泡也自动滚到可见区域
- 右栏：如果有关联的答题条目，也自动滚到可见区域

## 四、答题系统

### 4.1 支持的题型

| 题型 | 交互方式 | 判分显示 |
|------|----------|----------|
| **单选题** | 点选一个选项 | 正确选项旁 ✓（绿），错选旁 ✗（红） |
| **多选题** | 点选多个选项 | 同上 |
| **填空题** | 用户可不输入就直接提交 | 提交后空位中直接显示正确答案 |
| **连线题**（完形填空/七选五） | 将选项拖拽到对应空位（只显示选项序号ABCD，不显示内容） | 空位分前后两段：前段 ✓✗ 判分；若答错，后段浮现正确选项；若答对，后段不再重复显示正确选项 |

### 4.2 答案数据存储

正确答案由 AI 生成时写在 `data-*` 属性中：

```html
<!-- 选择题 -->
<div class="qa-option" data-option="A" data-correct="true">...</div>

<!-- 填空题空位 -->
<span class="qa-blank-slot" data-blank-id="1" data-correct-answer="however">___</span>

<!-- 连线题空位 -->
<span class="qa-blank-slot" data-blank-id="1" data-correct-answer="C">
  <span class="qa-blank-user">___</span>
  <span class="qa-blank-answer"></span>
</span>
```

### 4.3 判分结果动画

- ✓ 和 ✗ 标记以 `scale(0.3) → scale(1)` 的弹入动画渐入
- ✓ 绿色，✗ 红色
- 填空题的正确答案以 `opacity: 0 → 1` 渐入

### 4.4 答题与批注的隔离规则（重要）

**核心规则**：如果右栏（答题区域）内有答题内容，组件初始状态下**所有批注相关 UI 完全隐藏**——假装这篇文章从未被批注过：

- 左栏正文区域的所有批注角标（`note-badge`）不显示
- 左栏正文区域的所有批注富文本格式（下划线/高亮/变色等）不显示
- 中栏批注面板不可展开、控制按钮呈灰色不可用状态
- 右栏答题条目上的角标（`note-badge`）也不显示

**显示条件**：用户提交答案后，所有批注相关 UI 同时浮现（渐入动画），解锁批注讲解功能。

**编辑模式例外**：编辑模式下始终显示所有批注，否则老师无法编辑。

**CSS 状态类**：
- `.has-quiz` — 有答题内容
- `.submitted` — 已提交答案
- 编辑模式由编辑器全局状态控制

## 五、沉浸式逃逸与空间策略

### 5.1 沉浸式逃逸（Immersive Override）

答题批注组件需要尽可能多的屏幕空间。通过 CSS `:has()` 伪类实现**零 JavaScript** 的自动判定：当 Zone 2 中包含 `.quiz-annotation` 时，自动隐藏 Zone 1 标题栏，释放其占用的垂直空间。

```css
/* 沉浸式逃逸引擎 */

/* 包含答题批注组件的幻灯片，隐藏 Zone 1 标题栏 */
.slide:has(.quiz-annotation) .slide-header {
  display: none !important;
}

/* Zone 1 消失后，Zone 2 上移，填满释放的空间 */
.slide:has(.quiz-annotation) .slide-content {
  margin-top: 0;
  flex: 1;
}
```

**关于 Logo**：右上角品牌 Logo（`.branding`）使用 `position: fixed` 定位，挂在 `<body>` 下面，完全不属于 Zone 1，因此**天然不受影响**，无需任何解耦操作。

**关于上方留白**：`.slide` 自身的 `padding: var(--slide-padding)` 提供了足够的顶部间距（大屏约 64px），已经足以避免与 fixed 定位的 Logo（高约 48px）重叠，无需额外处理。

### 5.2 非对称 Padding 策略

为消除"双重 Padding"问题（`.slide` 的 padding + `.slide-content` 的 padding + 组件内部 padding 三重叠加），**组件内部四个方向的外围 padding 全部清零**，由 `.slide` / `.slide-content` 统管外围间距。

组件内部仅保留**栏间分隔 padding**：

| 区域 | 上 | 下 | 左 | 右 | 说明 |
|------|----|----|----|-----|------|
| 左栏（正文） | 无 | 无 | **无** | **有** | 左侧、上下交给外层统管；右侧 padding 作为与中栏的视觉间距 |
| 中栏（批注） | 无 | 无 | 小间距 | 小间距 | 气泡内容的呼吸空间 |
| 右栏（答题） | 无 | 无 | **有** | **无** | 右侧、上下交给外层统管；左侧 padding 作为与中栏的视觉间距 |

### 5.3 适用范围

沉浸式逃逸不仅适用于答题批注组件，后续计划中的以下组件也将使用同样的机制：

- 总封面页
- 章节封面页
- 封底页
- 二维码页

这些组件统一挂载 `.component-fullscreen` 类名（或直接利用 `:has()` 匹配各自专属的组件类名），共享同一套沉浸式逃逸 CSS 规则。

## 七、批注的创建与关联（编辑模式专属）

### 7.1 创建批注

#### 触发方式

在编辑模式下，用户在**左栏（正文）或右栏（答题）**中选中文字后，选区旁弹出浮动工具条：

```
┌─────────────────────────────────────────────────────┐
│  添加批注   [A变色] [🖍高亮] [U̲下划线] [S̶删除线]     │
└─────────────────────────────────────────────────────┘
```

- **"添加批注"**：文字标签，不是按钮，起引导说明作用
- **四个格式按钮**：点击任意一个按钮 = **同时**对选中文字应用该格式 + 创建新批注
- 将"选格式"和"创建批注"合并为一步操作

#### 操作流程

1. 在编辑模式下，选中**左栏或右栏**中的一段文字
2. 选区旁弹出浮动工具条
3. 点击任意格式按钮（如 `[U̲下划线]`）
4. 选中文字被包裹为 `<span class="text-anchor">` 并应用下划线格式
5. 自动生成新的角标（取当前最大 `data-link` 编号 +1）
6. 中间栏批注面板自动添加一个空的气泡，光标自动聚焦进去等用户输入
7. 新建的批注自动关联**选中文字所在侧**（左侧或右侧），另一侧暂不关联

### 7.2 关联另一侧的交互

创建批注后，批注气泡只关联了一侧。如需**建立双向关联**，通过气泡上的动态按钮操作：

#### 进入关联模式

1. 在编辑模式下，hover 批注气泡，若气泡**尚未关联某一侧**，会显示对应的 `🔗 关联左侧` 或 `🔗 关联右侧` 按钮
2. 点击按钮后进入**关联模式**：
   - 目标栏（左栏或右栏）出现微弱的**呼吸高亮边框**，提示"请在此区域选中文字"
   - 非目标栏的选中操作被忽略
3. 在目标栏中选中一段文字后，弹出"建立关联"浮动工具条：

```
┌─────────────────────────────────────────────────────┐
│  建立关联   [A变色] [🖍高亮] [U̲下划线] [S̶删除线]     │
└─────────────────────────────────────────────────────┘
```

4. 点击任意格式按钮 → 选中文字被标记为锚点 + 建立与气泡的关联 + 退出关联模式

#### 退出关联模式

- 按 `Esc` 键 → 取消，退出关联模式
- 点击页面其他区域（非目标栏）→ 取消
- 成功建立关联后 → 自动退出

### 7.3 格式函数复用

浮动工具条里的格式按钮调用的是 `editor-rich-text.js` 中已有的同一套格式化函数，但**额外触发"创建批注"或"建立关联"的逻辑**。顶部编辑工具栏依然正常工作（加粗/斜体/字号等普通编辑功能）。

### 7.4 浮动工具条的消失时机

- 用户点击了格式按钮 → 完成操作 → 消失
- 用户点击页面其他位置（取消选区）→ 消失
- 用户修改了选区范围 → 重新定位到新选区

## 八、数据结构

### 8.1 原文锚点（左栏）

```html
<span class="text-anchor" data-link="note-03" data-step="2"
      style="text-decoration: underline; text-decoration-color: var(--accent-blue);">
  被批注的文字
  <sup class="note-badge">2</sup>
</span>
```

### 8.2 答题条目锚点（右栏）

```html
<!-- 一道题对应多个正文批注时，角标横向排列 -->
<div class="qa-question-item" data-question-id="q1">
  <span class="answer-anchor" data-link-answer="note-01"><sup class="note-badge">1</sup></span>
  <span class="answer-anchor" data-link-answer="note-03"><sup class="note-badge">3</sup></span>
  题目内容...
</div>
```

### 8.3 批注气泡（中栏）

```html
<div class="qa-note-bubble" data-link="note-03" data-link-answer="q1" data-step="2" draggable="true">
  <div class="qa-note-handle">
    <span class="qa-note-step">2</span>
  </div>
  <div class="qa-note-content" data-edit-id="s1-note-03">
    批注内容，老师的讲解...
  </div>
</div>
```

### 8.4 关联机制

- `data-link`：批注唯一标识（自增 ID，永不改变、永不递补），关联**左栏正文**的 `text-anchor`
- `data-link-answer`：关联**右栏答题**的条目（可选属性，纯正文批注则不需要）
- `data-step`：步进序号（由中间栏气泡的上下排列顺序决定，拖拽排序后由 JS 重算；左右两栏角标使用同一套序号）
- 原文格式以 inline style 存在于 `text-anchor` 上
- 删除批注时：JS 通过 `data-link` 找到原文锚点 → 清除 inline style → 解包 `<span>` → 删除角标 → 通过 `data-link-answer` 找到右栏答题锚点 → 删除对应角标 → 删除气泡 → 重算序号

### 8.5 预留字段

- `data-note-type`：批注分类标签（第一版不使用，预留给后续"分类步进"功能）

## 九、技术整合

### 9.1 步进系统整合

通过现有 `registerStepStrategy()` API 注册新策略，无需修改 `slides-runtime.js` 核心逻辑。

但需要**小幅改造** `stepForward()`/`stepBackward()` 以支持"一个组件多步"：现有组件（flip/collapse）都是"一个组件一步"，批注组件是"一个组件 N 步"（每个批注是一步）。`hasNextStep()` 接口已预留，只需在步进函数中增加多步检查逻辑。

### 9.2 滚轮拦截

组件内的可滚动区域使用 `data-scrollable` 属性标记，利用现有 `slides-runtime.js` 的智能滚轮拦截。

### 9.3 空格键

空格键用于组件内正文区域的页内滚动，利用现有 `handleSpaceKey()` 函数。

### 9.4 页面切换钩子

利用 `slides-runtime.js` 提供的 `addSlideChangeListener()` 钩子接口监听翻页事件，重置批注步进状态。**不使用 MutationObserver**（已验证会与翻页 class 切换产生冲突）。

### 9.5 导出行为

Ctrl+S 导出时，清除所有答题状态（用户选择、✓✗ 标记），恢复为空白试卷。批注数据保留。
