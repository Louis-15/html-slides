# HTML 幻灯片编辑系统 —— 架构演进与全景规划 (v4.0)

## 背景与演进

将幻灯片演示内编辑功能从单体的面条代码，经过**系统性模块化重构**后，升级为一个具备清晰边界、高内聚低耦合的现代化富文本协同架构体系。
同时，随着功能的叠加，原生 HTML 编辑不再仅仅局限于文字，我们引入了完整的**自由涂鸦系统（Doodle）**，形成“结构化 DOM 编辑”与“自由层 Canvas 绘制”的双轴模式。

---

## 核心实现说明 & 近期更新成果

### 1. EditorCore (模式总控 & 初始化)
- 负责建立在任何页面的第一道防线。
- **最新优化**：实现了工具栏 UI 被代码动态注入，不再依赖丑陋的内联 HTML 字符硬编码。并在触发编辑态时强行屏蔽翻页事件，最大程度保护编辑安全。

### 2. RichTextToolbar (富文本工具栏)
承载了极其复杂的浏览器选区范围（`Selection & Range`）处理。
- **最新技术突破（混合样式连带 Bug 终结）**：
  - **原生样式引擎反转**：摒弃了对文本容器的粗暴抹除。当元素同时存在 `underline` 和 `line-through` 且准备剥离其中之一时，系统会对其做**正则级的手术分离**，不再“殃及池鱼”。
  - **连带污染防波堤**：重写了 `_splitDecoSpanAtRange` 的触发逻辑，只针对“局部脱离完全包裹范围”的取消行为做 DOM 的物理切割，向外扩充大样式时交还给原生 `extractContents` 缝合。这彻底消灭了“AB取消下划线后扩充C时，C意外沾染下划线”的跨层级样式泄漏 Bug。
  - **顶标引擎（Ruby）**：拥有自动吸附跟随底色的原生 `<ruby>` 处理能力，并配置了灵活响应相对主字号（`0.65em`）的缩放架构。

### 3. BoxManager (元素结构化)
统一了全局各种原生、非原生悬浮可编辑块的生命周期。
- 使用 `PointerEvent` 一次性整合全终端输入。
- 设计了“隐形防护”：保证不会因为删除或抽空内部文字导致父接点彻底坍塌。

### 4. HistoryManager (历史系统)
- **减负方案**：仅在当次 Session / 内存驻留。
- 一改过去笨重的全局双向数组，现在变成了针对当前 `slideIndex` 优化的多栈联动结构。
- 保证即便你使用了重级别的 CSS 魔法改变了节点，时光倒流时也能利用统一 `rehydrate()` 恢复事件绑定。

### 5. PersistenceLayer (持久化引擎)
- 采用了更可靠的基于文章 `Document.Title` 哈希分离的前缀，保障多个本地教案同时编辑并不串扰。
- Ctrl+S 是最极致的剥离算法：不仅克隆节点，还要深层穿透洗刷因为拖拽产生的内部把手，把最纯净的静态网页还给教师。

## 新增：涂鸦画图模块 (Doodle 体系)

涂鸦功能（通过 `doodle-runtime.js`）已被完全集成，它代表这套系统的**交互演进第二极**：

### 架构设计
- **层级隔离**：在原有的 DOM 编辑树之上，覆盖了一层独立触发的 `<canvas>` 面板。通过 `pointer-events` 的按需切换，使得手指与手写笔的物理移动轨迹被拦截，而不触发底层的 HTML 拖拽逻辑。
- **组件构成**：
  - **画笔引擎**：支持动态压感模拟与路径平滑（平滑贝塞尔曲线优化）。
  - **工具盒**：调色盘、笔触粗细调节板、橡皮擦引擎、以及几何图形扩展支持。
  - **无边框体验**：当激活时，整个 PPT 屏幕化作白板。

### 协同性验证与待办
涂鸦层拥有和 `HistoryManager` （DOM 状态机）截然不同的录制栈，为了不占用极高的 HTML 文本量。未来需要关注的核心在于：
> 当用户执行整页 **Ctrl+S 导出** 时，Canvas 层上的点阵数组（或 Base64 URL）如何伴随原生地嵌合在导出后的 HTML 里并进行正确重置渲染。
